3. Mode: AGENT — onboard each monitored cluster in two steps:
{{- if not .Values.config.databaseUrl }}

   ⚠  Agent enrollment requires a DURABLE store. config.databaseUrl is empty, so
      this control plane runs in-memory: minting returns 503 and agents are
      rejected. Set config.databaseUrl (LOTSMAN_DATABASE_URL) before onboarding.
{{- end }}

   a) Mint a per-cluster enrollment token from this control plane (admin-gated):

        POST /api/v1/enrollment-tokens   {"cluster":"<name>"}
        # or, via the CLI:
        lotsman cluster-token generate <name> --api http://<control-plane>:8080

   b) Install lotsman-agent in that cluster with the minted token, pointing at:

        controlPlane.addr: {{ include "lotsman-control-plane.fullname" . }}.{{ .Release.Namespace }}.svc:{{ .Values.service.gatewayPort }}
        agentToken.value:  <the token from step a>

   The token is bound to that exact cluster name. For agents OUTSIDE this cluster,
   expose the gateway port ({{ .Values.service.gatewayPort }}) via a LoadBalancer/NodePort — the ClusterIP
   Service is reachable in-cluster only.
{{- end }}
